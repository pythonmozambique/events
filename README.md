# Eventos da Comunidades

Este repositório centraliza a coordenação, documentação e monitorização de impacto das comunidades. O objectivo é garantir a transparência organizacional, a memória institucional e a recolha de dados estruturados.

## Fluxo de Trabalho e Contribuição

A gestão de actividades é realizada através do sistema de Issues do GitHub. Para garantir a padronização dos dados, é recomendado utilizar o modelo disponíveis ao criar uma nova issue

### 1. Relatório de Impacto (Pós-Evento)
Num prazo máximo de 15 dias após o encerramento da actividade, deve ser aberta uma nova Issue utilizando o modelo "Relatório de Evento (Post-Event Report)".

## Guia de Utilização: Relatório de Evento

Para manter a integridade dos dados de investigação, siga estes passos:

1. Aceda ao separador **Issues** e clique em **New Issue**.
2. Selecione o botão **Get Started** junto a **Relatório de Evento (Post-Event Report)**.
3. Preencha o formulário conforme o exemplo de estrutura abaixo:

**Exemplo de Título:** [REPORT] Django Girls Maputo - 6ª Edição

**Exemplo de Conteúdo:**
* **Informação Geral:** Evento realizado no Centro de Informática da UEM.
* **Demografia:** 30 participantes (100% mulheres). 80% Iniciantes, 20% Intermédias.
* **Orçamento:** Receita de xx.000 MZN (Patrocínio Local) + Grant PSF.
* **Retrospetiva:** A sessão de Deploy precisa de mais 30 minutos na próxima edição.

## Contacto e Credenciais
O acesso a credenciais de plataformas oficiais e redes sociais é restrito à equipa de coordenação activa. Utilize o canal próprio para solicitar acesso ou reportar questões administrativas.
